# Chat API Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a backend-owned stateless chat API that accepts `messages[]`, calls OpenAI, returns a simple JSON reply, and leaves a clean seam for future database-query tools.

**Architecture:** Add a new `chat` module in `apps/api` with controller, service, DTOs, provider abstraction, OpenAI-backed provider implementation, and an empty tool-registry seam. Keep the frontend contract minimal and stable while isolating provider logic inside backend code.

**Tech Stack:** NestJS, TypeScript, Jest, axios, existing OpenAI env config
